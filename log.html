SerienRecorder Anleitung

Die Log-Datei des SerienRecorder

Was bedeuten all diese Hinweise und Meldungen darin?

Die Log-Datei des SerienRecorders (SR) dient idR zur Kontrolle und Fehlersuche, falls es mal ein "Fehlverhalten" des SerienRecorders gibt. Diese Datei befindet sich idR im Plugin-Verzeichnis auf der Box unter "/usr/lib/enigma2/python/Plugins/Extensions/serienrecorder/" und ist eine reine Text-Datei.

Da die "Meldungen und Hinweise" in der Log-Datei, trotz mehrfacher Umformulierungen, immer noch häufig für Verwirrung sorgen, obwohl viele Einträge NUR Hinweise und KEINE Fehlermeldungen sind, sollen in diesem Artikel diese Ausgaben näher erläutert werden.

Oft resultiert ein "Fehlverhalten" des SR daraus, das der Benutzer etwas eingestellt bzw. aktiviert/deaktiviert hat, was für den Fall "unangebracht" war.
Beispiele:

  • "Der SR timert die Wiederholung der Staffel 3 nicht, obwohl ich die Folgen alle aus der Datenbank gelöscht habe?"
    Der User hat vergessen, das er eigentlich nur die neuen Folgen ab Staffel 4 haben wollte und hatte dies schon lange vorher mal im Marker Menü so eingestellt.
  • Der SR timert die neuen Folgen der Serie nicht?
    Der User hat vergessen, das er mal irgendwann die Serie im Marker Menü auf einen bestimmten Sender eingeschränkt hatte und die neuen Folgen dieser Serie auf einem anderen Sender Erstausgestrahlt werden.


Nun der Aufbau der Log-Datei im einzelnen


Mit jedem Auto-Check wird eine neue Logdatei angelegt, in diese wird solange geschrieben bis ein weiterer Auto-Check ein neue Datei anlegt.
Deswegen startet jedes Log mit folgender (beispielhaften) Zeile:
---------' Starte Auto-Check um 14.12.2019 - 19:36:38 (auto)'---------
Dort wird ausgegeben wann der Auto-Check gestartet (und damit die Logdatei erstellt) wurde. Zusätzlich wird noch ausgegeben ob es sich um einen automatischen (auto) oder manuellen (manuell) Auto-Check handelt.
Wenn die TV-Planer E-Mail verarbeitet wird (TV-Planer auto) oder (TV-Planer manuell).

Es werden folgende Situationen beim Auto-Check geprüft und sollte etwas nicht zutreffen, werden folgende Meldungen ausgegeben:

    1. Gibt es Serien-Marker?
      Es sind keine Serien-Marker vorhanden - Auto-Check kann nicht ausgeführt werden.
      ---------' Auto-Check beendet '---------
    2. Gibt es zugeordnete Sender?
      Es wurden keine Sender zugeordnet - Auto-Check kann nicht ausgeführt werden.
      ---------' Auto-Check beendet '---------
    3. Besteht eine Verbindung zum Internet?
      Keine Verbindung ins Internet. Check wurde abgebrochen!!
      ---------' Auto-Check beendet ( Ausführungsdauer: 0.0 Sek.)'---------


Die Logdatei ist in Abschnitte gegliedert, worin nicht immer alle Informationen angezeigt werden, da sich manche über Logfilter (DEBUG LOG), in den SerienRecorder Einstellungen, an- und abschalten lassen.
Grundsätzlich gibt es zwei Arten von Logeinträgen, diejenigen die immer ausgegeben werden und diejenigen die nur bei aktiviertem Filter ausgegeben werden.

Für eine eventuelle "Fehlersuche" ist es aber SEHR von Vorteil, alle DEBUG LOG-Optionen aktiviert zu haben. Auch das "Backup von TV Planer e-Mail erstellen:" kann dabei helfen.
Das aktivieren des SR eigenen Backups bewirkt, das sowohl die Datenbank als auch die Timer der Box bei jedem Auto Check (idR tägl.) gesichert werden.


1. Abschnitt "Die Umgebung"


Hier wird angezeigt, in welcher "Umgebung", also Hard- und Software, der SerienRecorder seinen Dienst verrichtet.

  1. Box (STB) Informationen
    Z.B. "Uno 4K SE" oder "dm900"

  2. Aktivierte Log Filter
    Z.B. Timer, Debug etc.

  3. Konfigurierte Aufnahmeverzeichnisse
    Dies dient weniger der Übersicht der Verzeichnisse, sondern ist hauptsächlich dafür gedacht, um Netzlaufwerke (NAS) anzusprechen, um diese vor dem Auto-Check aus dem Standby zu "wecken".

  4. Aktion "SerienPlaner Daten laden"
    Daran kann man sehen, ob die Daten für die "Planer-Ansicht" korrekt geladen wurden.

  5. Anzahl zugeordneter Sender
    Es kommt immer wieder mal vor, das User nur 2-3 Sender zuordnen und sich dann wundern, wenn keine Timer erstellt werden.

  6. Berücksichtigte Zeitspannen beim Auto-Check
    Auch die Zeitspanne wird gerne außer Acht gelassen bei der Frage "Warum wird für nächsten Freitag die Serie XY nicht getimert".

3. Abschnitt "Daten vom SerienServer verarbeiten"


In diesem Abschnitt werden nur Hinweise und/oder Fehlermeldungen ausgegeben.
Während dieses Verarbeitungsschrittes werden die Ausstrahlungstermine vom SerienServer abgerufen und bereits grob eingeschränkt.

Je nachdem ob die TV-Planer Option aktiv ist oder nicht, ändert sich die Kopfzeile.


  1. Es wird ein voller Suchlauf durchgeführt - dies ist die Standardmeldung wenn die TV-Planer Option nicht aktiv ist.
    ---------' Verarbeite Daten vom Server - voller Suchlauf' ---------
  2. Die TV-Planer Option ist aktiv, aber es wurden keine TV-Planer Daten gefunden - es wird ein voller Suchlauf durchgeführt.
    ---------' Verarbeite Daten vom Server - keine TV-Planer Daten - voller Suchlauf' ---------
  3. Die TV-Planer Option und die Option "Voller Suchlauf mindestens einmal im Erstellungszeitraum" sind aktiv und die Zeit für einen vollen Suchlauf ist erreicht.
    ---------' Verarbeite Daten vom Server - Zeit abgelaufen - voller Suchlauf' ---------
  4. Die TV-Planer Option ist aktiv und die Option "Timer für Filme anlegen" ist deaktiviert - es werden nur Timer für Serien erstellt.
    ---------' Verarbeite Daten vom Server - nur Serien der TV-Planer E-Mail' ---------

Mögliche Hinweis- bzw. Fehlermeldungen in diesem Abschnitt:

    1. Es sollen für diesen Serien-Marker keine Ausstrahlungstermine vom SerienServer abgerufen werden, weil die Timer nur aus den Daten der TV-Planer E-Mail angelegt werden sollen.
      Für diesen Serien-Marker sollen nur Timer aus der E-Mail angelegt werden.
    2. Die Meldung taucht dann auf, wenn es ein Problem beim Abrufen der Ausstrahlungstermine vom SerienServer gab. Das ist eine allgemeine Fehlermeldung, wenn etwas beim Abrufen schief gelaufen ist. Es kann sich also um einen Serverfehler, Server nicht erreichbar, Daten beschädigt etc. handeln.
      Solche Fehler sind meist innerhalb von ein paar Stunden (idR spätestens am nächsten Tag) wieder weg.
      Fehler beim Abrufen und Verarbeiten der Ausstrahlungstermine
    3. Wenn die Option "Immer Timer anlegen, wenn keine Wiederholung gefunden wird" deaktiviert ist wird die Meldung ausgegeben, dass sich die Sendung nicht in der konfigurierten Zeitspanne befindet.
      (DEBUG LOG - Tageszeit)
      Sendung (AUSSTRAHLUNGSZEIT) nicht in Zeitspanne [KONFIGURIERTE AUSSTRAHLUNGSZEIT]
    4. Wird ausgegeben, wenn ein Sender nicht gefunden wurde, falsche oder fehlende Senderzuordnung.
      (DEBUG LOG - Senderliste)
      STB-Sender nicht gefunden ' -> ' SENDER '
    5. Wird ausgegeben, wenn ein Sender deaktiviert ist
      (DEBUG LOG - Senderliste)
      STB-Sender deaktiviert -> ' SENDER '
    6. Wird ausgegeben, wenn die Episode nicht erlaubt ist - also am Serien-Marker Einschränkungen für Episoden vorgenommen wurden
      (DEBUG LOG - Episoden)
      Episode nicht erlaubt -> ' EPISODE ' -> ' ERLAUBTE EPISODEN '
    7. Wird ausgegeben, wenn die Staffel nicht erlaubt ist - also am Serien-Marker Einschränkungen für Staffeln vorgenommen wurden
      (DEBUG LOG - Episoden)
      Staffel nicht erlaubt -> ' STAFFEL ' -> ' ERLAUBTE STAFFEL '
    8. Falls es für die Episode einen Eintrag auf dem Merkzettel gibt
      Timer vom Merkzettel wird angelegt @ SENDER


4. Abschnitt "Daten aus TV-Planer E-Mail verarbeiten"


In diesem Verarbeitungsschritt werden die Ausstrahlungstermine aus der TV-Planer E-Mail ausgelesen und grob eingeschränkt.
Die Ausgabe der Hinweis- und Fehlermeldungen entspricht denen aus Abschnitt 3.

    Beispielhafter Log-Eintrag

    ---------' Verarbeite Daten aus TV-Planer E-Mail '---------

    ' American Chopper - S03E14 - Bike zu versteigern 2 ' - Staffel nicht erlaubt -> ' S03E14 ' -> ' [1] '

6. Abschnitt "Aktualisiere Timer"


Während dieses Verarbeitungsschrittes werden bereits erstellte Timer aufgrund von aktuellen Daten vom SerienServer oder des EPGs aktualisiert.

    Beispielhafter Log-Eintrag

    ---------' Aktualisiere Timer '---------

    ' FBI: Special Crime Unit - S01E21 - Rote Seide - /media/hdd/movie/ '
    Startzeit wurde aktualisiert von 13.12. - 02:35 auf 13.12. - 02:30
    ' Helden der Baustelle - S01E10 - Anpfiff auf der Baustelle - /media/hdd/movie/ '
    Event ID wurde aktualisiert von 0 auf 60583
    ' Gute Zeiten, schlechte Zeiten - S00E6910 - Folge 6910 - /media/hdd/movie/ '
    Beschreibung wurde aktualisiert von S00E6910 - Folge 6910 auf S00E6910 - Überraschende Unterstützung
    ' Aktenzeichen XY...ungelöst - S00E549 - Folge 549 - /media/hdd/movie/ '
    Timer konnte nicht aus dem EPG aktualisiert werden, da der Abgleich fehlgeschlagen ist @ ZDF


Die "DEBUG LOG" Ausgaben im einzelnen


In den Einstellungen des SerienRecorder kann man festlegen welche Arten von Logzeilen beim Auto-Check ausgeben werden sollen.
Um eine kompakte Übersicht über die erstellten und aktualisierten Timer, beim Auto-Check, zu bekommen reicht es den Filter "Timer Debugging" zu aktivieren.
Alle anderen Logfilter geben noch zusätzliche Informationen aus und vereinfachen die Fehlersuche.

Die folgende Liste zeigt welche Informationen mit welchem Filter ausgegeben werden:

    1. STB Informationen

      Gibt Informationen über die Box, das verwendete Image bzw. dessen Version, die SerienRecorder bzw. Datenbank Version und die Skinauflösung (der Box) aus.
      Die Ausgabe von "STB Type" ist nicht immer korrekt, weil besonders die open* Images die auf den unterschiedlichsten Boxen laufen, nicht immer den richtigen Typ ausgeben.

      STB Type: dm900
      Image: 4.3.2r7-17-g32f8eb-2019-08-30 / Experimental 2018-02-22
      SR Version: 3.9.7-beta
      Datenbank Version: 3.9.6
      Skin Auflösung: 1280 x 720
